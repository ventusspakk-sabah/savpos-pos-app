# Sav.pos 獨立應用程序架構文檔 (Independent 3-App Architecture)

本專案已按照您的要求，正式拆分為三個獨立運行的應用程序。雖然它們在視覺風格上保持一致且數據互通，但每個應用都有其獨立的入口點、導航邏輯與權限驗證。

---

## 🧖 1. Sav.pos POS (實體店營運端)
**目標用戶：** 前台接待、收銀員、店鋪經理。
- **獨立入口：** `/app/pos`
- **核心流程：** 
    - 樓層/房間監控與開單。
    - 技師現場調度。
    - 多重支付結帳與自動抹零。
    - 訂單歷史與退款處理。
- **導航：** 底部導覽包含「樓層」、「銷售」、「訂單」、「更多」。

---

## 💆 2. Sav.pos Tech (技師個人端)
**目標用戶：** 按摩師、理療師、技師。
- **獨立入口：** `/app/tech`
- **核心流程：**
    - 個人收益與 30 天帳期佣金明細。
    - 服務狀態切換（開始/結束）與客戶簽名。
    - 報銷、假勤與預支申請中心。
    - 個人排班表查看。
- **導航：** 底部導覽包含「首頁」、「收益」、「申請」、「個人」。

---

## 👔 Sav.pos Admin (老闆管理端)
**目標用戶：** 投資人、店長、財務主管。
- **獨立入口：** `/app/admin`
- **核心流程：**
    - 全局經營儀表板與業績對比分析。
    - 財務審批中心（報銷、佣金批量支付、沖銷）。
    - 數據核對中心（Google Drive 備份與對帳）。
    - 系統級配置（權限矩陣、PIN 碼、服務目錄）。
- **導航：** 底部導覽包含「看板」、「審核」、「設定」、「日誌」。

---
**技術說明：** 三個 App 共用一套後端 API 與數據庫，確保數據即時同步。在部署時，可分別打包為三個獨立的 PWA 應用。
