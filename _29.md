# Sav.pos 系統壓力測試 (Stress Test) 計劃書

## 1. 測試目標
驗證系統在高併發環境下的穩定性，特別是針對：
- **佣金計算引擎**：模擬 1000+ 筆訂單同時結算，確保 30 天帳期保護與沖銷邏輯無誤。
- **資料庫寫入**：高頻率的庫存扣減與訂單狀態更新。
- **通知隊列**：大量結帳成功後的簡訊/郵件自動發送。

## 2. 壓力場景模擬
- **高峰期模擬**：10 分鐘內湧入 500 筆訂單。
- **併發操作**：多名收銀員同時對同一熱門服務進行結帳。
- **異常注入**：模擬網路波動導致的支付狀態回調延遲。

## 3. 監控指標
- **平均回應時間 (Latency)**：需維持在 200ms 以內。
- **計算準確率 (Integrity)**：佣金與總營收數據必須 100% 匹配。
- **隊列積壓數 (Backlog)**：通知發送延遲不得超過 5 秒。

## 4. 驗證流程
1. 啟動模擬腳本。
2. 實時監測「壓力測試儀表板」。
3. 抽查 10% 的隨機訂單佣金明細。
4. 生成壓力測試報告與優化建議。
